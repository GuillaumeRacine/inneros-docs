# File Paths

Standard file paths and directory conventions used throughout InnerOS. Understanding this structure is essential for creating agents, commands, and integrations that reference the right locations.

## Core Paths

| Path | Purpose | Type |
|------|---------|------|
| `~/Obs_Vault/` | Obsidian vault root (symlink to shared vault) | Directory |
| `~/.claude/` | Claude Code configuration directory | Directory |
| `~/.claude/agents/` | Agent definition files (symlink) | Directory |
| `~/.claude/commands/` | Slash command definitions (symlink) | Directory |
| `~/.claude/hooks/` | Pre/post tool execution hooks (symlink) | Directory |
| `~/.claude/scripts/` | Utility scripts (symlink) | Directory |
| `~/.claude/skills/` | Multi-step skill packs (symlink) | Directory |
| `~/.claude/settings.json` | Global permissions and hook config | File |
| `~/.claude/settings.local.json` | Machine-specific overrides | File |
| `~/.claude/CLAUDE.md` | Global context (read every session) | File |
| `~/CLAUDE.md` | Project-level context for home directory | File |
| `~/.secrets` | API keys and environment variables (chmod 600) | File |

## Symlink Mapping

The `~/.claude/` directories are symlinks to the vault's `_System/Claude_Code/` directory. This means the vault is the single source of truth, and changes in either location are reflected in the other.

| Local Path | Vault Source |
|------------|-------------|
| `~/.claude/commands/` | `~/Obs_Vault/_System/Claude_Code/Commands/` |
| `~/.claude/agents/` | `~/Obs_Vault/_System/Claude_Code/Agents/` |
| `~/.claude/hooks/` | `~/Obs_Vault/_System/Claude_Code/Hooks/` |
| `~/.claude/scripts/` | `~/Obs_Vault/_System/Claude_Code/Scripts/` |
| `~/.claude/skills/` | `~/Obs_Vault/_System/Claude_Code/Skills/` |

### Setting Up Symlinks

```bash
# Remove existing directories (back up first if they have content)
rm -rf ~/.claude/commands ~/.claude/agents ~/.claude/hooks ~/.claude/scripts ~/.claude/skills

# Create symlinks
ln -s ~/Obs_Vault/_System/Claude_Code/Commands ~/.claude/commands
ln -s ~/Obs_Vault/_System/Claude_Code/Agents ~/.claude/agents
ln -s ~/Obs_Vault/_System/Claude_Code/Hooks ~/.claude/hooks
ln -s ~/Obs_Vault/_System/Claude_Code/Scripts ~/.claude/scripts
ln -s ~/Obs_Vault/_System/Claude_Code/Skills ~/.claude/skills
```

### Settings Files (Copied, Not Symlinked)

Some files are copied rather than symlinked because Claude Code expects them in specific locations:

| File | Why Not Symlinked |
|------|-------------------|
| `~/.claude/settings.json` | Claude Code reads this from a fixed path |
| `~/.claude/settings.local.json` | Machine-specific, should not sync |
| `~/.claude/CLAUDE.md` | Fixed path, read at session start |

When updating these files, edit the vault copy first, then copy to `~/.claude/`.

## External Storage

For large files, project workspaces, and sensitive data that should not be in the synced vault:

| Path | Purpose | Sync |
|------|---------|------|
| `[EXTERNAL]/0_System/` | Private vault, sensitive documents | None |
| `[EXTERNAL]/0_System/Obsidian_Private/` | Private Obsidian vault (credentials, identity docs) | None |
| `[EXTERNAL]/1_Projects/` | Active project workspaces (large repos, builds) | Git per-project |
| `[EXTERNAL]/3_Resources/` | Reference materials (large files) | None |
| `[EXTERNAL]/4_Archives/` | Completed projects, historical data | None |

## Database Locations

Local databases accessed by data source scripts:

| Database | Path | Access Mode |
|----------|------|-------------|
| Photos.sqlite | `~/Pictures/Photos Library.photoslibrary/database/Photos.sqlite` | Read-only (`immutable=1`) |
| WhatsApp | `~/Library/Group Containers/group.net.whatsapp.WhatsApp.shared/ChatStorage.sqlite` | Read-only (`immutable=1`) |
| Voice Memos | `~/Library/Group Containers/group.com.apple.VoiceMemos.shared/Recordings/` | Read-only (file access) |

**Important:** All database access uses `immutable=1` SQLite mode to prevent WAL lock conflicts with the host applications.

## Path Variables

For use in scripts and agent context files, these shorthand variables map to full paths:

| Variable | Resolves To |
|----------|------------|
| `$INNEROS` | `~/Obs_Vault/0_InnerContext` |
| `$CONTEXT` | `~/Obs_Vault/0_InnerContext/Self_Context` |
| `$COACHES` | `~/Obs_Vault/0_InnerContext/Coaching/Coaches` |
| `$WORKFLOWS` | `~/Obs_Vault/0_InnerContext/Workflows` |
| `$WRITERS` | `~/Obs_Vault/0_InnerContext/Workflows/Writing/Agents` |
| `$KAI` | `~/Obs_Vault/_System` |
| `$CLAUDE_CODE` | `~/Obs_Vault/_System/Claude_Code` |

These are reference conventions, not necessarily exported shell variables. Use them in documentation and agent context sections.
