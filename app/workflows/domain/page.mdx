# Domain-Specific Workflows

Beyond cadence-based workflows, InnerOS has specialized workflows for each life domain. These workflows have domain-specific tools, data sources, and agent expertise. Nine domains are currently active.

| Domain | Agents | Key Workflows | Primary Data Sources |

| DeFi/Trading | 8 | Position monitoring, rebalancing, yield tracking | DeFi protocols, price feeds |
| Writing | 13 | Research, drafting, publishing, distribution | Obsidian vault, CMS, analytics |
| Music | 4 | Practice tracking, recording, production, release | DAW, streaming platforms |
| Home | 3 | Maintenance scheduling, contractor management, projects | Calendar, task lists |
| Health | 6 | Workout logging, sleep analysis, medication, nutrition | Wearables, health apps |
| Productivity | 12 | Focus sessions, time tracking, inbox triage, reviews | Calendar, git, Claude Code |

| Family | 4 | Communication tracking, scheduling, therapy coordination | Calendar, messages |
| Learning | 7 | Course progress, research, spaced repetition | Course platforms, notes |

## DeFi and Trading

The most automated domain, with agents monitoring positions around the clock.

### Position Monitoring

```yaml
name: defi-position-monitor
frequency: Every 15 minutes (automated)
agents: dn-monitor, yield-tracker

workflow:
  - Fetch current positions from on-chain data
  - Calculate unrealized P&L
  - Check positions against risk thresholds
  - Alert if any position needs attention
  - Log snapshot to tracking database

alerts:
  - Position value drops > 5% in 24 hours
  - Yield drops below minimum threshold
  - Impermanent loss exceeds configured limit
  - Gas costs exceed yield for the period
```

### Weekly DeFi Review

```bash
# Triggered via skill
/dn

# Output: Dashboard showing
# - Active positions with current values
# - Week-over-week yield
```

## Writing

The most agent-heavy domain with a full 13-stage pipeline. See the [Writing Pipeline](/writing-pipeline) section for complete documentation.

### Key Workflows

```yaml
# Research workflow
research:
  trigger: New article idea captured
  steps:
    - Researcher agent gathers sources
    - Synthesizer extracts patterns
    - Narrative Architect proposes structure
    - Output: Research brief in Drafts/

# Publishing workflow
publish:
  trigger: /publish-prep
  steps:
    - Final review by Style Compression agent
    - Format for target platform (Substack, blog, etc.)
    - Generate social media excerpts
    - Schedule publication
    - Archive to canon after publishing

# Distribution workflow
distribute:
  trigger: After publication
  steps:
    - Cross-post to configured platforms
    - Generate newsletter version
    - Update content index
    - Track engagement metrics
```

## Music

Music workflows bridge digital tools with physical practice.

### Practice Session Workflow

```yaml
name: music-practice
trigger: Calendar block or manual start
duration: 60-90 minutes

steps:
  - name: Setup
    type: automated
    action: Load previous session notes, pull up practice plan

  - name: Warm-up
    type: human
    duration: 10 min
    content: Scales, finger exercises (from practice plan)

  - name: Focused practice
    type: human
    duration: 40-60 min
    content: Current piece or technique goal

  - name: Log session
    type: agent
    prompt: |
      Record what was practiced, what improved,
      what needs more work. Update practice plan.

  - name: Track progress
    type: automated
    action: Update practice streak, log duration
```

### Recording Session Workflow

```yaml
name: recording-session
trigger: Scheduled block (typically evening)
duration: 2-3 hours

steps:
  - Review project status and previous session notes
  - Set specific goal for this session (1-2 tasks only)
  - Execute recording/production work
  - Export session files, update project notes
  - Log what was accomplished and next steps
```

## Adding Domain Workflows

To add a workflow for a new domain:

```bash
# 1. Create domain folder
mkdir -p 0_InnerContext/Workflows/NewDomain/

# 2. Create workflow definition
cat > 0_InnerContext/Workflows/NewDomain/workflow_name.md << 'EOF'
name: workflow-name
domain: new-domain
cadence: weekly
...
EOF

# 3. Register in workflow index
echo "| workflow-name | new-domain | weekly | manual |" >> WORKFLOW_INDEX.md

# 4. Create agent if needed (see /customization/your-agents)
```
