# Prerequisites

InnerOS runs on macOS with a specific set of tools. Verify each one before proceeding to installation.

## Required Software

| Tool | Minimum Version | Purpose |
|------|----------------|---------|
| macOS | 14.0 (Sonoma) | Operating system (Apple Silicon recommended) |
| Claude Code | Latest | Agent runtime, command execution |
| Obsidian | 1.5+ | Knowledge base, vault management |
| Node.js | 20+ | Script execution, automation pipelines |
| Python 3 | 3.11+ | Data fetcher scripts, NightCrew stages |
| Git | 2.40+ | Version control, repo sync |

## Verify Your Environment

Run each command to confirm the tool is installed and meets the minimum version:

```bash
# macOS version
sw_vers --productVersion
# Expected: 14.0 or higher

# Claude Code
claude --version
# Expected: any recent version

# Obsidian (check if installed)
ls /Applications/Obsidian.app
# Expected: should exist

# Node.js
node --version
# Expected: v20.x or higher

# Python 3
python3 --version
# Expected: 3.11 or higher

# Git
git --version
# Expected: 2.40 or higher
```

Or run the one-liner to check everything at once:

```bash
echo "macOS: $(sw_vers --productVersion)" && \
echo "Claude: $(claude --version 2>/dev/null || echo 'NOT FOUND')" && \
echo "Node: $(node --version 2>/dev/null || echo 'NOT FOUND')" && \
echo "Python: $(python3 --version 2>/dev/null || echo 'NOT FOUND')" && \
echo "Git: $(git --version 2>/dev/null || echo 'NOT FOUND')" && \
echo "Obsidian: $(ls /Applications/Obsidian.app > /dev/null 2>&1 && echo 'installed' || echo 'NOT FOUND')"
```

## Installing Missing Tools

### Claude Code

Claude Code is Anthropic's official CLI for Claude. Install it globally via npm:

```bash
npm install -g @anthropic-ai/claude-code
```

You will need an Anthropic API key or a Claude Pro/Max subscription. See the [Claude Code documentation](https://docs.anthropic.com/en/docs/claude-code) for setup details.

### Obsidian

Download from [obsidian.md](https://obsidian.md). The free version is sufficient. Obsidian Sync is optional but recommended if you want to access your vault from mobile devices.

### Node.js

Install via the official installer or a version manager:

```bash
# Using n (recommended)
curl -fsSL https://raw.githubusercontent.com/tj/n/master/bin/n | bash -s lts

# Or using Homebrew
brew install node
```

### Python 3

macOS includes a system Python, but you may want a newer version:

```bash
# Check if system Python is sufficient
python3 --version

# If not, install via Homebrew
brew install python@3.12
```

### Git

Git ships with Xcode Command Line Tools:

```bash
xcode-select --install
```

## Optional Tools

These are not required for the core system but enable specific features:

| Tool | Purpose | Install |
|------|---------|---------|
| 1Password CLI | Secure credential storage for agents | `brew install --cask 1password-cli` |
| GitHub CLI (`gh`) | GitHub API access for agents | `brew install gh` |
| jq | JSON processing in scripts | `brew install jq` |
| Homebrew | Package manager for optional tools | [brew.sh](https://brew.sh) |

### 1Password CLI

If you plan to use data source integrations (email, calendar, health APIs), you will need a way to manage API keys. InnerOS supports 1Password CLI as the primary credential store, with fallback to environment variables and `.env` files.

```bash
# Install
brew install --cask 1password-cli

# Verify
op --version

# Sign in (requires 1Password desktop app with CLI integration enabled)
op signin
```

### GitHub CLI

Used by several agents for repository management, issue tracking, and project board access:

```bash
brew install gh
gh auth login
```

## Hardware Recommendations

InnerOS is designed for a single-machine setup. The recommended configuration:

| Component | Recommendation | Why |
|-----------|---------------|-----|
| Machine | Mac mini / MacBook (Apple Silicon) | Consistent always-on environment |
| RAM | 16GB minimum | Multiple Claude Code sessions |
| Internal SSD | 256GB+ | OS, vault, Claude Code config |
| External SSD | 1TB+ (always connected) | Projects, archives, large files |

The external drive is not strictly required, but InnerOS's storage strategy assumes one is available for projects and archives. See [Storage Strategy](/architecture/storage-strategy) for details.

## Environment Variables

Create a secrets file that your shell sources on startup:

```bash
# ~/.secrets (chmod 600)
export ANTHROPIC_API_KEY="your-key-here"
# Add other API keys as needed for data sources
```

Add to your shell config:

```bash
# In ~/.zshrc
source ~/.secrets
```
