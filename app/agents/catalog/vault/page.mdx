# Vault & Knowledge Management

Agents for vault curation, linking, archival, and Obsidian integration.

**7 agents** in this domain.

| Agent | Description |
|-------|-------------|
| `link-validator` | Scans configuration files for file paths and validates they exist. Reports broken links that would break workflows. Use on schedule or after renames. |
| `naming-auditor` | Audits and recommends file/folder renames to match universal naming conventions. Enforces numbered PARA structure at top level, snake_case everywhere else. Produces recommendations for approval - neve |
| `obsidian-cli` | Obsidian CLI power user - knows every command across official CLI (v1.12+) and notesmd-cli. Handles all vault CRUD, search, automation, and scripting via terminal. |
| `vault-curator` | Obsidian vault data quality agent - audits, cleans, organizes, tags, formats, and archives notes. Ensures every note is properly structured, linked, and classified within PARA. |
| `vault-dashboard` | Renders the Meta Vault command center view |
| `vault-guardian` | Enforces vault location rules, prevents duplication, and maintains structural integrity of the shared PARA vault. |
| `vault-linker` | Knowledge graph agent - builds connections, MOCs, backlink analysis, link suggestions, and maintains the vault's internal linking structure. |
